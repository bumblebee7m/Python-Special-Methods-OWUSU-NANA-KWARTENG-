# Python Special Methods (Magic/Dunder Methods) in OOP

## Overview

This repository contains practical examples of Python Special Methods (also known as Magic or Dunder Methods) covered in the EL 162/234 Object Oriented Programming course.

The project demonstrates how Python's special methods allow user-defined classes to behave like built-in Python objects through constructors, string representations, operator overloading, indexing, iteration, and more.

---

## Topics Covered

- `__init__()` – Constructor
- `__str__()` – User-friendly string representation
- `__repr__()` – Developer representation
- `__len__()` – Define object length
- `__eq__()` – Equality comparison
- `__add__()` – Addition operator overloading
- `__sub__()` – Subtraction operator overloading
- `__mul__()` – Multiplication operator overloading
- `__getitem__()` – Item access
- `__setitem__()` – Item assignment
- `__contains__()` – Membership testing
- `__call__()` – Callable objects
- `__iter__()` and `__next__()` – Iteration support
- `__bool__()` – Boolean evaluation
- `__del__()` – Destructor

## Learning Outcomes

After completing this notebook, you should be able to:

- Understand the purpose of Python special methods.
- Create constructors using `__init__()`.
- Customize object representations with `__str__()` and `__repr__()`.
- Overload operators such as `+`, `-`, and `*`.
- Implement indexing and item assignment.
- Make custom objects iterable and callable.
- Perform membership testing and boolean evaluation.
- Apply multiple special methods in a practical ShoppingCart project.
